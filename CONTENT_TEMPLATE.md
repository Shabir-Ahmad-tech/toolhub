# Tool Page Content Brief (for writer or AI-assisted draft)

Use this exact 5-part structure for every tool page. 400-800 words total. Write for a developer audience — assume technical literacy, skip basic explanations of what JSON/regex/etc. is in general, focus on how THIS tool solves a specific pain point better than alternatives.

## Structure (same order every time)

1. **What it is** (2-3 sentences) — what the tool does, who uses it, the specific problem it solves.
2. **How it works** — the actual logic/format/spec involved (e.g. cron syntax fields, JWT structure, regex flags). Real technical detail, not fluff.
3. **Worked example** — one real, concrete example with actual input/output shown.
4. **Common mistakes** — 2-3 specific errors developers actually make with this task.
5. **FAQ** — 3-5 short Q&A pairs, real questions developers would search for.

## Rules

- Every tool's content must be genuinely unique — no reused sentence structures across tools.
- Before writing, check what currently ranks #1 on Google for the target query and note one specific gap this page fills that the top result doesn't.
- No filler, no generic SEO padding ("In today's digital world...").
- No regional/country references anywhere.

## Owner's review checklist (10 min per tool, do not skip)

- [ ] Matches the 5-part structure
- [ ] Technical accuracy is correct (owner is the domain expert — verify the formula/logic is actually right)
- [ ] Worked example numbers/output are correct
- [ ] No filler sentences
- [ ] FAQ questions sound like real searches, not made up
